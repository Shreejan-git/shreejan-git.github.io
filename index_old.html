scAInit

Document Intelligence · VLM · Production AI

A VLM-based financial document understanding system for extracting structured information from checks, invoices, bank statements, credit card statements, and tax reports.

My Role: ML Lead / AI Engineer

Problem: Financial documents contain inconsistent layouts, tables, dense text, and varied formats, making reliable extraction difficult.

What I Built: Designed and deployed a document understanding pipeline with dataset versioning, annotation QA, benchmark evaluation, and AI microservice integration.

Evaluation: 0.825 BLEU, 0.127 CER, 0.312 WER, 0.805 TEDS, 0.849 structure-only TEDS on a 376-image annotated benchmark dataset.

Impact: Passed EFT/UAT for production release.

Python FastAPI vLLM DVC S3 MongoDB EC2

ChatWithDoc

RAG · Document QA · Layout-Aware Retrieval

A layout-aware document QA/RAG system designed for asking source-grounded questions over complex financial documents.

My Role: RAG System Architect

Problem: Standard chunking and retrieval methods struggle with financial documents containing tables, sections, multi-page context, and layout-dependent information.

What I Built: Designed contextual chunking, Qdrant-based vector retrieval, reranking, and grounded LLM generation for source-aware answers.

Evaluation: Established retrieval evaluation using Precision@k, Recall@k, Hit@k, and source-grounding checks to compare chunking, embedding, reranking, and retrieval configurations.

Impact: Improved reliability of question answering over complex documents by making retrieval and answer generation more traceable and source-grounded.

LlamaIndex Qdrant RAG Reranking LLM Grounded Generation

TelePlantDoctor

Computer Vision · Mobile AI · Vector Search

A crop disease diagnosis mobile AI application developed for FAO-related agricultural use cases.

My Role: AI Backend / Computer Vision Engineer

Problem: Farmers and agriculture stakeholders needed a practical mobile-based system to identify crop diseases from field images.

What I Built: Developed AI microservices for disease detection, feature-based matching, and diagnosis support using computer vision and vector similarity search.

Impact: Enabled AI-based crop disease diagnosis workflow through mobile application integration.

FastAPI Faster R-CNN MobileNetV3 Milvus Cosine Similarity Mobile AI

HemScreen

Healthcare AI · Detection · Segmentation

An AI mobile application for non-invasive anemia screening using lower eyelid, thumbnail, and palm images.

My Role: Computer Vision Engineer

Problem: Non-invasive anemia screening requires identifying and analyzing relevant visual regions from mobile-captured body-part images.

What I Built: Created detection and segmentation pipelines using Faster R-CNN with MobileNetV3 and U-Net.

Impact: Supported a mobile AI workflow for health screening using computer vision-based region extraction.

Faster R-CNN MobileNetV3 U-Net OpenCV Segmentation Mobile AI

In-house Annotation Platform

Internal Tooling · Data Pipeline · Annotation QA

A custom annotation management platform built to organize, distribute, track, and validate data annotation work for ML projects.
